# 极简 UI/UX 交互设计 SOP (基于第一性原理)

**核心理念 (First Principles)**:
1.  **Don't Make Me Think**: 用户只想买完东西回家，不想研究地图。
2.  **Zero Friction**: 能 1 步完成的，绝不让用户点 2 次。
3.  **最小改动**: 复用现有界面架构，只加“点睛之笔”。

---

## 🗺️ 地图篇："顺路买" (On the Way)
**目标**: 在不改变现有操作习惯的前提下，让用户无感使用“顺路”导航。

### 1. 交互流程 (The Flow)
*   **现状**: 列表页点击 `Navigate` -> 地图页显示最近超市。
*   **优化后**: 列表页点击 `Navigate` -> 地图页**自动**推荐“最顺路”超市。

### 2. UI 改动点 (Minimal Changes)

#### A. 设置“家/公司” (一次性配置)
*   **入口**: `Profile` 或 `Settings`。
*   **动作**: 用户输入 Home/Work 地址。
*   **原理**: 有了锚点，才能计算“顺路”。

#### B. 地图展示页 (`ShoppingRouteScreen`)
不用加新页面，只改底部那个白色卡片 (`infoCard`)。

*   **Header 区**:
    *   增加一个**小图标** (🏠/🏢)：点击切换“终点是家”或“终点是公司”。
*   **底部卡片 (`infoCard`) 进化**:
    *   **原显示**: "Distance: 2.5km"
    *   **新显示**: "🚗 **+3 mins** detour to Home" (去家仅多绕路 3 分钟)
    *   **视觉重点**: 用绿色高亮 "+3 mins"，这是用户决策的唯一依据。
*   **地图图层**:
    *   画出：`当前位置` -> `超市` -> `家` 的整条蓝色轨迹。
    *   让用户一眼看到：“哦，确实是顺路的”。

### 3. 给开发者的 SOP
1.  **检测**: 进入页面时，检查是否有保存的 Home 地址。
2.  **计算**: 若有，静默调用 API 计算 `Current -> Store -> Home` 的时间。
3.  **展示**: 哪怕用户没去点任何按钮，直接在底部卡片显示 “顺路指数” (Detour Time)。
4.  **排序**: 如果有多个分店，自动选中那个“绕路时间最短”的，而不是“物理距离最近”的。

## 🗣️ 语音篇："一键直达" (Press & Speak)
**目标**: 像微信发语音一样简单，随时随地记东西。

### 1. 交互流程 (The Flow)
*   **现状**: 无入口。
*   **优化后**: 在所有主页面右下角悬浮一个“麦克风”按钮。

### 2. UI 改动点 (Minimal Changes)

#### A. 全局悬浮球 (Floating Action Button)
*   **位置**: 屏幕右下角 (Thumb Zone，拇指最舒适区域)。
*   **样式**: 圆形，品牌色背景，白色麦克风图标 🎙️。
*   **交互逻辑 (极致简单)**:
    1.  **按住 (Press & Hold)**: 手机震动一下，开始录音。
    2.  **说话**: "买了牛奶，想吃番茄炒蛋，删掉面包。"
    3.  **松手 (Release)**: 录音结束，显示 "Thinking..." Loading 动画。
    4.  **反馈**: 顶部弹出 Toast 提示 "✅ 已更新清单：+牛奶 +番茄炒蛋 -面包"。

### 3. 给开发者的 SOP
1.  **不跳转**: 千万别跳转到新页面去录音，打断用户流。就在当前页面处理。
2.  **乐观 UI**: 松手瞬间先给一个“收到”的反馈，后台 API 慢慢跑，跑完再弹结果通知。
3.  **容错**: 如果没听清，弹窗问一句“是这个意思吗？”，只有一个“Yes”按钮和一个“重录”按钮。

---

## 总结：用户怎么用？
1.  **买东西前**：按住右下角说“今晚想吃牛排”，松手即完成。
2.  **下班路上**：点一下 `Navigate`，看到提示“去这家 Sainsbury's 只绕路 3 分钟”，直接开车走人。

**这就是“对用户最方便”的设计：少思考，少点击。**
